## 📖 API & Web
> Application Programming Interface(API)는 **소프트웨어간의 통신 규칙**이다.

### 1. API를 테스트 하는 도구
|도구|설명|
|--|--|
|Postman|GUI 기반. API 호출, 테스트 스크립트, 자동화 가능. 엔터프라이즈 환경에서도 많이 사용|
|Insomnia|Postman보다 가볍고, 특히 GraphQL 테스트에 편리|
|cURL|터미널에서 명령어로 API 호출. 스크립트에 쉽게 넣을 수 있음|
---
### 2. API 통신 규격의 진화 (SOAP -> REST -> GraphQL/gRPC)
> API 규격도 **생산성**과 **효율성**을 위해 변화해옴

| API 규격  | 주요 특징                  | 데이터 형식     | HTTP 버전          | 장점                | 단점                             |
| ------- | ---------------------- | ---------- | ---------------- | ----------------- | ------------------------------ |
| SOAP    | 엄격한 프로토콜, 트랜잭션/보안 강력   | XML        | 주로 HTTP/1.1      | 보안과 신뢰성 높음        | 무겁고 느림                         |
| REST    | 리소스 중심, 직관적            | JSON / XML | HTTP/1.1, HTTP/2 | 간단하고 구현 용이        | 오버페칭 발생 가능                     |
| GraphQL | 클라이언트가 필요한 데이터 선택      | JSON       | HTTP/1.1, HTTP/2 | 데이터 최적화, 프론트엔드 자유 | 학습 곡선 존재, 캐싱 복잡                |
| gRPC    | 함수 호출(RPC), 내부 MSA 최적화 | Protobuf   | HTTP/2           | 빠르고 효율적           | 바이너리 포맷 디버깅 어려움, 브라우저 직접 호출 제한 |

---
### 3. HTTP/1.1 vs HTTP/3
| HTTP 버전 | 핵심 특징  | 장점 | 단점 |
| ---| --- | --- | --- |
| **HTTP/1.1** | 텍스트 기반, 요청마다 TCP 연결 | 단순, 대부분 환경 지원| 연결 오버헤드, 동시 요청 처리 느림 |
| **HTTP/2**   | 멀티플렉싱, 헤더 압축, 서버 푸시| 속도 빠름, 지연시간 감소, 서버 푸시 가능 | 구현 복잡, 일부 오래된 환경 미지원 |
| **HTTP/3** | UDP 기반, 연결 지연 최소화, QUIC 프로토콜 사용 | 빠른 연결, 끊김 방지, 모바일 친화 | 지원 범위 제한, 구현 복잡|

---
### 4. URL의 구조 & 엔드포인트 설계
`https://api.example.com/v1/users?id=123`
| 구조                | 설명                  |
| ----------------- | ------------------- |
| `https://`        | 프로토콜 (HTTP / HTTPS) |
| `api.example.com` | 도메인                 |
| `v1`              | 버전 관리 (Versioning)  |
| `users`           | 리소스 이름              |
| `?id=123`         | 쿼리 파라미터 (특정 리소스 식별) |

---
### 5. JSON vs XML
|특징|JSON|XML|
| --- | ---| ---|
|형식| Key-Value 기반|태그 기반|
|가독| 사람이 읽기 쉬움|비교적 무겁고 복잡|
|용량| 작음| 크고 반복적|
|지원| 웹, JavaScript 친화적 | SOAP 등에서 많이 사용|
|장점| REST API에 최적| 복잡한 스키마, 트랜잭션, 보안에 강력|
> GraphQL과 REST에서는 JSON을 사용
